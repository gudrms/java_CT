## 🔥 라이브 코딩 테스트 (14:00-16:00, 2시간)

### 📌 공통 특징
- **문제 난이도**: 과제보다 어려움, 프로그래머스 Lv.2 수준
- **시간**: 2-3시간 (실제로는 2시간 고정)
- **환경**: 구름IDE 또는 현장 컴퓨터
- **평가 기준**: 요구사항 충족 + 코드 품질 + 자료구조 선택 근거

### 🚨 실제 출제 경향
1. **알고리즘 구현 문제** (백준 골드 1-3 수준)
2. **자료구조 선택 이유 설명** (왜 Vector? 왜 HashMap?)
3. **시스템 설계 문제** (대용량 트래픽 처리)

### ⚠️ 주의사항 (후기에서 발췌)
> "라이브코딩은 요구 사항에 맞게 프로그램을 구현하는 시간이며, **체감상 과제보다 어려웠음**"

> "코딩테스트의 코드를 보면서 질문을 받았는데, 입력받은 데이터를 **벡터에 담았는데, 왜 벡터에 담았는지 해시맵이나 리스트를 사용할 생각은 안했는지** 질문 받았음"

> "**수백만명이 접속해서 경쟁적으로 예약을 받아야 하는 콘서트 티켓 예약 시스템을 설계**할 때 어떤 식으로 할 것인지 묻는 질문"

### ✅ 대응 전략
1. **자료구조 선택 시 항상 이유 메모**
   - ArrayList vs LinkedList
   - HashMap vs TreeMap
   - Stack vs Queue

2. **코드 작성하면서 주석으로 설명**
   ```java
   // HashMap 사용: O(1) 조회 성능 필요
   HashMap<String, Integer> map = new HashMap<>();
   ```

3. **시스템 설계 문제 대비**
   - 대용량 트래픽 → 로드밸런서, 캐싱, DB 샤딩
   - 동시성 → Lock, Queue, Redis
